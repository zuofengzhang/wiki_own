JS 同源策略

请参考，了解一下同源策略

同协议，domain（或ip），同端口视为同一个域，一个域内的脚本仅仅具有本域内的权限，可以理解为本域脚本只能读写本域内的资源，而无法访问其它域的资源。这种安全限制称为同源策略。 ( 现代浏览器在安全性和可用性之间选择了一个平衡点。在遵循同源策略的基础上，选择性地为同源策略"开放了后门"。 例如img script style等标签，都允许垮域引用资源。)

下表给出了相对 http://store.company.com/dir/page.html 同源检测的示例:

URL 结果 原因
http://store.company.com/dir/other.html 成功
http://store.company.com/dir/inner/another.html 成功
https://store.company.com/secure.html 失败 协议不同
http://store.company.com:81/dir/etc.html 失败 端口不同(默认80)
http://news.company.com/dir/other.html 失败 主机名不同

由于浏览器同源策略的限制，让我们无法通过js直接在不同的域之间进行数据传输或通信，比如用ajax向一个不同的域请求数据，或者通过js获取页面中不同域的框架(iframe)中的数据。

JSON跨域及实现方案

利用<script>元素允许访问跨域引用资源的特点

  1. 在当前页面上建立一个js回调函数(如jsCallback函数)
  2. 使用<script src="">..引入不同域的js，并将本地的回调函数(如jsCallback)以参数的形式传递过去。
  3. 在不同域的js中实现获取回调函数名称，执行访问本地资源的操作，并将执行结果以json的形式发回回调函数(如jsCallback("{"abc":123}"))

由于在调用远程的js与本地的js相同，当js调用回调函数时，就可以调用了。而本地的回调函数需要的就是远程以json形式传进来的数据。

利用DOM的特性可以动态的生成回调函数，将远程访问和回调函数组合起来。就是AJAX做的事情。

AJAX

ajax函数(dataType:jsonp)

$.ajax({
    type: 'post',
    url: "http://127.0.0.1:9000/jsonp_post.php",
    crossDomain: true,
    data: {u: 'alsy', age: 20},
    dataType: "jsonp",
    success: function(r){
        console.log(r);
    }
});

get

//$.getJSON()方法
$.getJSON("http://127.0.0.1:9000/jsonp.php?callback=?", {u:'abc', p: '123'}, function(jsonData){
    console.log(jsonData);
});
